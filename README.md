# キーワード分析レポート生成ツール - 実行手順書

## 概要
StanbyADのキーワードデータから分析レポートを生成するツールです。

## データの取得元

### 1. キーワードレポートのローデータ
**databricksから取得**
- アカウントIDと期間を指定して取得
- 取得したデータをExcelファイルとして保存

### 2. キャンペーン別の合計データ
**シャーレのキャンペーンレポートから取得**
- キャンペーンレポートからCSVをダウンロード
- ファイル名に日付範囲を含む必要があります（例: `20260101-20260131`）
- ファイル名に`Campaigns`を含む必要があります
- 前期間と後期間の2つ以上のファイルが必要です

## セットアップ（初回のみ・必須）

### ステップ1: Pythonのインストール（必須）
このツールを実行するためには、**Python（パイソン）**というプログラム言語の実行環境が必要です。

#### Pythonとは
- データ分析などで使われるプログラム言語です
- このツールを動かすために必要な基盤ソフトウェアです

#### インストール方法

**macOSの場合：**
1. ブラウザで以下のURLを開く：
   https://www.python.org/downloads/
2. 「Download Python 3.x.x」（最新版）ボタンをクリック
3. ダウンロードしたファイル（例: `python-3.x.x-macos.pkg`）をダブルクリック
4. インストーラーの指示に従って進める
   - 「続ける」をクリック
   - 「インストール」をクリック
   - パスワードを入力（求められた場合）
5. インストールが完了したら、以下の手順で確認：
   - ターミナル（アプリケーション > ユーティリティ > ターミナル）を開く
   - 以下のコマンドを入力してEnterキーを押す：
     ```bash
     python3 --version
     ```
   - `Python 3.x.x`のようにバージョンが表示されればOK

**Windowsの場合：**
1. ブラウザで以下のURLを開く：
   https://www.python.org/downloads/
2. 「Download Python 3.x.x」（最新版）ボタンをクリック
3. ダウンロードしたファイル（例: `python-3.x.x-amd64.exe`）をダブルクリック
4. インストーラーで**必ず「Add Python to PATH」にチェックを入れる**
5. 「Install Now」をクリック
6. インストールが完了したら、以下の手順で確認：
   - コマンドプロンプト（Windowsキー + R → `cmd`と入力してEnter）を開く
   - 以下のコマンドを入力してEnterキーを押す：
     ```bash
     python --version
     ```
   - `Python 3.x.x`のようにバージョンが表示されればOK

**注意事項：**
- Python 3.9以上が必要です（3.9, 3.10, 3.11, 3.12など）
- すでにインストールされている場合でも、上記の確認コマンドでバージョンを確認してください
- インストールに時間がかかる場合があります（5〜10分程度）

### ステップ2: ツールのダウンロードと展開
1. このツールのフォルダを任意の場所に配置してください（例: デスクトップ）
2. フォルダ名は`keywordanalysis`のままでOKです

### ステップ3: 初回実行時の自動セットアップ
1. **macOSの場合**: `実行.command`をダブルクリック
2. **Windowsの場合**: ターミナル（コマンドプロンプト）で以下のコマンドを実行：
   ```bash
   cd [ツールのフォルダのパス]
   python run.py
   ```
3. 初回実行時は、必要なライブラリ（追加のプログラム）が自動でインストールされます
   - インターネット接続が必要です
   - 数分かかる場合があります
4. 2回目以降は通常の実行のみです（自動セットアップはスキップされます）

## 実行手順（2回目以降）

### ステップ1: データの準備
以下のディレクトリ構造でデータを配置してください：

```
keywordanalysis/
├── data/
│   ├── 後期間/
│   │   └── [databricksから取得したExcelファイル]
│   ├── 前期間/
│   │   └── [databricksから取得したExcelファイル]
│   └── totals/
│       └── [シャーレから取得したCampaigns.csvファイル（2つ以上）]
└── output/
    └── [生成されたレポート]
```

### ステップ2: データの配置
1. **キーワードレポートのローデータ**
   - databricksから取得したExcelファイルを、`data/後期間/`または`data/前期間/`に配置
   - 期間に応じて適切なフォルダに配置してください

2. **キャンペーン別の合計データ**
   - シャーレからダウンロードしたCSVファイルを、`data/totals/`に配置
   - 前期間と後期間の2つ以上のファイルが必要です
   - ファイル名には日付範囲（例: `20260101-20260131`）と`Campaigns`を含めてください

### ステップ3: 実行
#### macOSの場合
1. `実行.command`をダブルクリック
2. ターミナルが開いて処理が実行されます
3. 完了後、Enterキーを押して終了

#### Windowsの場合（またはmacOSでターミナルから実行）
1. ターミナル（コマンドプロンプト）を開く
2. プロジェクトディレクトリに移動：
   ```bash
   cd [ツールのフォルダのパス]
   ```
   例: `cd C:\Users\あなたの名前\Desktop\keywordanalysis`
3. 以下のコマンドを実行：
   ```bash
   python account_analyzer.py
   ```

### （任意）キャンペーン合算機能について
- 実行中に「合算するキーワード条件」を聞かれた場合、入力したキーワードを含むキャンペーンを合算します（例: `看護師,介護`）。
- **合算を指定した場合、サマリーの「キャンペーン別の指標」は合算後キャンペーン単位で前後比較**されます（Campaigns.csvの合計値も同じルールで合算して比較します）。
- 何も入力せずにEnterを押すと、合算は行われません（従来どおりキャンペーン単位）。 

### 注意（`run.py` について）
- `run.py` は**キャンペーン合算に非対応**です（合算ONで実行すると `_2` シート複製やCV/平均CPAの不整合が起きるため、現在はエラー停止します）。\n+  合算する場合は `アカウント分析くん.command` もしくは `python run_grouping.py` を使ってください。

### ステップ4: 結果の確認
- `output/`ディレクトリにExcelファイルが生成されます
- ファイル名: `[アカウント名]_KWレポート.xlsx`
- 生成されたファイルを開いて内容を確認してください

## 出力レポートの内容

生成されるExcelファイルには以下のシートが含まれます：

1. **サマリーシート**
   - アカウント全体の指標
   - キャンペーン別の指標とシェアの変化
   - 影響の大きいキーワードTOP5

2. **アカウントシート**（前期間・後期間別）
   - COST降順、CV降順、CPA昇順で3パターン表示
   - CPA列に条件付き書式（平均からの乖離を色分け）

3. **キャンペーンシート**（各キャンペーンごと、前期間・後期間別）
   - アカウントシートと同様の構成
   - キャンペーン単位のCPA平均で条件付き書式を適用

## トラブルシューティング

### Pythonがインストールされているか確認できない
- ターミナル（コマンドプロンプト）で`python3 --version`（macOS）または`python --version`（Windows）を実行
- バージョンが表示されない場合は、Pythonのインストールが完了していません
- 上記の「ステップ1: Pythonのインストール」を再度確認してください

### ライブラリのインストールに失敗する
- インターネット接続を確認してください
- ファイアウォールやセキュリティソフトがブロックしていないか確認してください
- それでも解決しない場合は、以下のコマンドを手動で実行してみてください：
  ```bash
  pip3 install -r requirements.txt
  ```
  （Windowsの場合は`pip install -r requirements.txt`）

### データファイルが見つからない
- `data/`ディレクトリ配下のフォルダ構造を確認してください
- ファイル名が正しいか確認してください（日付範囲、`Campaigns`を含むなど）
- フォルダ名が正確か確認してください（`後期間`、`前期間`、`totals`）

### エラーが発生した場合
エラーが発生した場合は、ターミナル（コマンドプロンプト）に表示されたエラーメッセージをそのままコピーして、武田にSlackで連絡してください。
